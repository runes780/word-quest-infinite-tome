# 学习科学、游戏化与用户体验提升路线图

> 审计日期：2026-07-15
> 范围：本地优先学习入口、今日路径、战斗答题与反馈；不引入账号、云同步或真实学习者数据。

## 1. 设计判断

Word Quest 已经具备提取练习、FSRS、掌握度、错题补救、学习证据和游戏反馈。下一阶段的重点不是继续堆叠金币、动画或随机奖励，而是让每个游戏反馈都服务于以下学习循环：

1. **先提取，再反馈**：让学习者主动作答；答案之后给出能解释证据的反馈，而非只显示对错。
2. **按记忆风险安排复习**：以 FSRS 到期、近期错误和掌握证据组织今日路径，避免纯随机刷题。
3. **把支架逐步撤掉**：提示用于修复，不应永久替代回忆；接近掌握后使用新语境迁移检查。
4. **奖励学习进步，不奖励无意义点击**：奖励应绑定完成提取、纠正错因、到期复习和迁移成功。
5. **保留自主感**：学习者能看懂为什么推荐这条路径，也能切换到复习卡片或每日挑战。
6. **控制额外认知负荷**：动画、战斗特效和术语不能遮挡题干、答案、解释与下一步行动。

研究依据：测试效应支持用提取练习而非重复阅读促进延迟保持；间隔效应表明复习间隔需要与目标保持时间匹配；带反馈的测试尤其能修复错误与低把握的正确答案。参考 [Roediger & Karpicke, 2006](https://www.psychologicalscience.org/journals/psychological-science/j.1467-9280.2006.01693.x/)、[Cepeda et al., 2006](https://pubmed.ncbi.nlm.nih.gov/16719566/) 和 [Butler et al., 2008](https://pubmed.ncbi.nlm.nih.gov/18605878/)。游戏元素应按其支持的心理需要和学习行为选择，而不是假定“游戏化本身有效”；参考 [Sailer et al., 2017](https://doi.org/10.1016/j.chb.2016.12.033)。元分析显示游戏化对认知、动机和行为结果总体为小幅正效应，但动机/行为结果的稳健性较弱、异质性较高；参考 [Sailer & Homner, 2020](https://link.springer.com/article/10.1007/s10648-019-09498-w)。最新的自适应提取练习实验也提示点数与进度条可以提高主观能力感和愉悦度，却不必然提高延迟回忆，因此本项目只把奖励作为可解释反馈，不把奖励数值当作学习效果；参考 [Gamified feedback in adaptive retrieval practice, 2026](https://www.sciencedirect.com/science/article/pii/S0747563225003097)。

## 2. 本轮审计结论

| 优先级 | 问题 | 学习/体验影响 | 决策 | 状态 |
|---|---|---|---|---|
| P0 | 无 API Key 时自动弹出设置，主按钮写成“配置 API 密钥以开始” | 阻断首次提取练习，错误暗示学习依赖外部 AI，也削弱本地隐私承诺 | 本地练习成为主入口；AI 仅作为自有材料生成题目的可选增强 | 已实现 |
| P0 | 从首页底部进入战斗后保留旧滚动位置 | 学习者看不到战斗状态和题目起点 | 任务切换时回到顶部 | 已实现 |
| P1 | 今日路径虽有步骤理由，但未解释证据边界和可改选性 | 容易把推荐误解为强制安排或高风险判断 | 明示“依据本机学习记录”，保留复习/挑战选择 | 已实现 |
| P1 | 答案解释在移动端可能只露出一部分 | 反馈被奖励动画和页面长度遮挡 | 答题后将反馈卡以最小位移滚入可见区 | 已实现 |
| P1 | 状态主要依赖颜色/动画，图标按钮名称不完整 | 屏幕阅读器、键盘和减少动态效果用户难以确认状态 | 增加状态播报、可访问名称、44px 关键目标和 reduced-motion | 已实现 |
| P1 | 学习者把握度未进入答题证据 | 无法区分“猜对”“稳定掌握”和“高把握误解” | 先做低摩擦原型与事件字段评审，再进入学习数据契约 | 已实现 |
| P2 | 奖励仍较多绑定单题命中与战斗经济 | 可能鼓励速度/刷题，而非到期复习和错因修复 | 每题 XP/金币改由练习、独立提取、补救、到期复习和迁移证据决定；战斗分数/连击不再放大奖励 | 已实现 |
| P2 | 家长面板指标多，但行动优先级仍重 | 证据可能被误作最终判断 | 以“一项证据 + 一项可执行建议 + 置信边界”重排 | 后续里程碑 |

## 3. 分阶段实现

### M0：零阻力、本地优先学习入口（已实现）

- 首次进入不自动打开设置。
- 无密钥时可直接启动内置合成练习；明确不会调用 AI。
- “连接 AI”降为次要动作，仅用于基于自有资料生成题目。
- 今日路径解释推荐依据并允许改选。
- 任务切换重置滚动位置，反馈卡保持可见。
- 关键状态使用 `aria-live`，图标按钮有名称，核心触控目标至少 44px。
- Framer Motion 和 CSS 动画均尊重系统 reduced-motion 偏好。

成功指标：首次学习入口可见率、首次任务启动率、本地入口启动耗时、无密钥设置弹窗率（应为 0）。当前项目没有真实用户遥测，因此只定义指标，不声称效果提升。

### M1：元认知校准与解释性反馈（已实现）

1. 仅在选择题形式的诊断题和迁移题上提供可选的“没把握 / 有点把握 / 很有把握”，不选择也能直接作答。
2. 将作答前自评把握度作为 `learningEvents.selfConfidence` 可选字段；它不是索引字段，因此旧记录、旧备份和现有 Dexie schema 保持兼容。
3. 对“高把握错误”和“低把握正确”显示不同复盘提示，但不改变分数、奖励、FSRS 评级、技能掌握度或学习目标掌握度。
4. 本轮报告和家长/教师视图只显示聚合校准信号，不显示把握度对应的原题，也不做能力标签或排名。

验收：字段缺失向后兼容；battle/SRS/daily 事件契约保持一致；高把握错误和低把握正确触发差异化反馈；AI 报告请求不新增该字段；导出仍不含原题和自有材料。

### M2：把游戏奖励迁移到学习进步（已实现）

1. 纯函数按“错因修复 → 到期提取 → 新语境迁移 → 无提示独立提取 → 支架练习”对正确答案分类；强证据获得更高的基础 XP 和金币。
2. 同一题面只发放一次进步奖励；支架练习、独立提取、补救、到期复习和迁移各有单局上限，且全局最多 6 次。超限练习仍保留学习事件，但奖励为 0 并显示原因。
3. 暴击和连击继续影响战斗伤害与分数，但不再提高每题 XP/金币；遗物和祝福倍率仍在通过防刷规则后作用于合格奖励。
4. 战斗反馈直接说明“完成支架练习 / 独立提取 / 错因修复 / 到期记忆 / 迁移成功”及实际 XP/金币，不只显示数值跳动。
5. 每次实际发放与防刷结果写入本地 `learningEvents` 可选字段；任务报告和家长面板仅显示聚合证据，不显示奖励对应的原题。
6. 移除任务结束时对复习、薄弱点和迁移证据的第二次货币发放，避免同一证据重复结算；mastery 状态升级的一次性奖励保持独立。

验收：纯函数、battle/SRS/补救集成、备份兼容、AI 输入排除和浏览器持久化均有测试；奖励规则不改变 FSRS 评级或 mastery；重复题、同类证据与单局总量均无法无限刷取；所有实际奖励可回溯到学习事件。

### M3：支架淡出与迁移闭环（已实现）

1. 答题事件现在记录该次答案是否实际使用提示，并与单独的提示打开事件保持一致；旧记录缺少该字段时仍按“未知/未记录”兼容，不新增 IndexedDB 索引。
2. 可测试纯函数以同一学习目标最近 6 次答案为窗口：一次正确只保持当前支架；同一级别至少两次无提示成功才淡出一级；较少支架下两次成功才进入迁移准备。
3. 高一级支架的历史成功不能用于跳过新一级的证据门槛；即时补救成功也不会被当成撤支架证据，下一题仍保留部分支持再确认。
4. 迁移准备只会提前题包中已经通过内容质量门的迁移题，不会把普通同语境题重新标记成迁移题；若当前题包没有合格迁移题，则保持独立练习而不伪造证据。
5. 普通错误插入一次不同题面的即时补救；补救题错误不再连续插入补救。迁移失败回到带支架练习，但反馈明确说明这只是下一步支持信号，不是能力定论。
6. 每次路径决策把 `scaffoldTransition`、`scaffoldReason`、`nextSupportLevel` 和 `nextAttemptKind` 作为本地可选事件字段保存；战斗反馈用文字解释原因。
7. 任务报告和家长/教师面板同时展示带支架练习、独立提取、提示使用、淡出/补救和迁移证据；只展示聚合计数，不显示对应原题或学习材料。

验收：纯函数、同目标路径、提示一致性、battle 集成、报告聚合、备份兼容和 AI 输入排除均有回归测试；支架只在两次同级无提示成功后撤除；迁移失败不会直接降为能力定论；路径变化均有受控原因码和学习者可读解释。

研究边界：支架的最优给/撤时机仍属于“帮助困境”，即时表现更好不等于长期学习更好，参见 [Koedinger et al., 2008](https://www.cs.cmu.edu/~bmclaren/pubs/KoedingerEtAl-IsItBetterToGiveThanToReceive-CogSci2008.pdf)。2025 年专长反转元分析显示低先备知识者受益于较高帮助，但低龄和语言学习样本中的撤支架效应更不确定，因此本实现选择保守的两次同级成功门槛，参见 [Tetzlaff et al., 2025](https://doi.org/10.1016/j.learninstruc.2025.102142)。迁移证据使用真正的新语境/新题型而非同题复现；提取练习迁移的综合效应及其对初始成功、精加工反馈的依赖参见 [Pan & Rickard, 2018](https://pdf.retrievalpractice.org/transfer/Pan_Rickard_2018.pdf)。

### M4：真实试点前的体验研究

- 使用合成或公开材料完成教师认知走查、键盘走查、375/768/1024/1440 视口测试。
- 真实学习者试点必须另行完成隐私、同意、数据最小化和退出机制评审。
- 预注册主要指标：延迟复测、到期复习完成、提示依赖、迁移表现；游戏时长只作辅助指标。

## 4. 体验守则

- 题干、答案、解释和下一步永远比装饰动画优先。
- 正误不能只靠红/绿表示，必须有文字或图标语义。
- 重要移动端控件以 44px 为产品目标，并至少满足 WCAG 2.2 的目标尺寸/间距要求。
- 动态反馈可增强能力感，但必须支持减少动态效果，不得用持续动画制造紧迫感。
- 任何 AI 输出都可由教师或监护人复核；服务失败时回退到安全本地内容。
- 指标是学习支持证据，不是最终评价、诊断或排名。

可访问性依据见 W3C 对 [目标尺寸](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html)、[交互动画](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions.html) 和 [状态消息](https://www.w3.org/WAI/WCAG22/Understanding/status-messages.html) 的说明。
