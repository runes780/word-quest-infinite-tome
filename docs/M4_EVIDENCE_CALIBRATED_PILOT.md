# M4：证据校准与真实试点准备

状态：仓库内实现与自动化验证已完成；真实学习者试点尚未开始，也不得从本地测试推断学习成效。

## 1. 产品判断

Word Quest 的下一阶段不应继续增加更多积分、徽章或题型，而应先回答三个更基础的问题：

1. 系统声称学习者“会了什么”时，题目是否真的测量同一个、已注册的知识成分？
2. 一次正确是带支架练习、即时修复、独立提取、延迟保持，还是新语境迁移？
3. 游戏奖励是否服务于学习路径，而不是把重复点击、连续天数或即时正确伪装成学习成效？

本里程碑采用 Knowledge–Learning–Instruction 的知识成分粒度、Evidence-Centered Design 的“主张—证据—任务”关系，并把提取练习、延迟保持、迁移和高信息反馈落实为可审查的数据契约。研究支持设计假设，不代表本产品已经产生同等效果。

## 2. 已实现的产品闭环

### 2.1 学习目标与题目契约

- Objective Registry v2 为目标提供稳定 ID、版本、别名、先修关系、知识成分类型和最低证据要求。
- 未注册目标进入隔离状态：可以保留练习历史，但不能更新目标掌握度。
- 每道题记录题目族、语境、等价组、测量角色、迁移距离和审核状态。
- 普通练习、即时修复、延迟探测和迁移不再共享一个模糊的“正确次数”。

### 2.2 证据强度与掌握度

- `supported`：带明显支架、使用提示或即时修复；可反馈进步，不足以证明保持。
- `independent`：注册目标上的独立作答。
- `delayed-independent`：到期后、作答前不展示反馈的独立保持探测。
- `transfer-independent`：审核过的新语境/新题面迁移任务。
- `no-credit`：未知目标、未审核测量题或不满足证据契约的尝试。

目标掌握度保留所有尝试用于历史审计，但只有合格证据推动巩固/掌握状态。少于 3 次合格尝试时，家长面板隐藏百分比分数；“mastered”必须同时有延迟与迁移证据。FSRS 负责“何时再见”，目标掌握度负责“证据支持到什么程度”，二者不再互相替代。

### 2.3 24 小时、7 天与迁移路径

- 只有来自审核内容的独立基线才会产生 24 小时与 7 天保持探测。
- 探测使用同一等价组的不同题面/语境，不重复原题冒充保持。
- 错误先进入带支架修复，并至少冷却 24 小时后再测；即时修复成功不计作延迟保持。
- 受控本地内容包覆盖全部注册目标的练习、24 小时与 7 天等价变体，并有答案键与语境唯一性测试。

### 2.4 成人审核与 AI 边界

- AI 题默认不具备迁移/保持测量资格。
- 教师或监护人可在本地审核队列中批准或拒绝；批准只改变题目资格，不代表学习者掌握。
- 审核记录与题目缓存以一个 IndexedDB 事务更新；审核内容不会被发送给 AI。
- 数据库 v15 新增审核表；v14 升级时旧“mastered”会保守降为“consolidated”，新的证据计数从 0 开始，避免把旧口径伪装成新证据。

### 2.5 反馈、游戏化与体验

- 正误同时使用图标、文字、正确答案、“为什么”和可迁移策略，不只依赖红/绿。
- 奖励区分支持练习、独立成功、修复、延迟保持和迁移；未审核内容不发强证据奖励。
- 修复奖励表达努力与策略调整，但不计入强证据总数。
- “连续火焰”改为“活跃学习日 + 自选目标”；休息不会抹去掌握证据，也不使用损失框架逼迫回访。
- 重要新控件采用至少 44px 目标；减少动态效果偏好会停止非必要缩放、过渡与持续动画。

## 3. 研究依据与保守解释

- [KLI framework](https://pubmed.ncbi.nlm.nih.gov/22486653/) 强调知识成分、学习事件和教学事件应匹配，因此系统按目标而非泛化“技能分”聚合证据。
- [Evidence-Centered Design](https://onlinelibrary.wiley.com/doi/10.1002/j.2333-8504.2003.tb01908.x) 要求任务观察与学习主张之间有明确证据论证，因此题目角色和证据强度被持久化并可审计。
- [学校情境提取练习系统综述](https://doi.org/10.1007/s10648-021-09595-9) 支持在真实课堂中研究提取练习，但不能据此把一次即时正确解释为长期保持。
- [提取练习迁移元分析](https://pubmed.ncbi.nlm.nih.gov/29733621/) 表明迁移可能发生且受边界条件影响，因此迁移题必须使用新语境并独立统计。
- [教育反馈元分析](https://www.frontiersin.org/journals/psychology/articles/10.3389/fpsyg.2019.03087/full) 显示反馈总体有效但异质性很高，且控制性、低信息反馈可能损害动机；因此反馈提供答案、原因和策略，不做人格判断。
- [学习游戏化元分析](https://link.springer.com/article/10.1007/s10648-019-09498-w) 支持同时观察认知、动机与行为结果，也显示情境和社会互动是重要调节项；因此 XP、连续日和游戏时长都不能作为主要学习指标。
- [WCAG 2.2 目标尺寸](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html) 与 [交互动画](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions.html) 指导触控目标和 reduced-motion 实现。

## 4. 教师认知走查（可立即执行）

只使用仓库合成材料，每位走查者完成以下任务并边做边说：

1. 从首页解释“今日路径为什么推荐这些步骤”。
2. 区分练习、24 小时保持、7 天保持和新语境迁移。
3. 审核一条 AI 题，说明批准意味着什么、不意味着什么。
4. 从面板回答“哪个目标已有保持证据、哪个仍缺延迟或迁移证据”。
5. 导出报告前复述包含/排除字段和隐私风险。

记录任务成功、首次误解、犹豫点、严重度与建议，不记录姓名、学校或真实学生信息。阻断性问题必须在真实试点前清零；文案偏好不与证据逻辑混为一谈。

## 5. 真实学习者试点协议（尚未执行）

### 5.1 开始门槛

- 获得适用地区所需的组织/伦理审查、监护人知情同意与学习者同意或认可。
- 明确退出、删除本机数据、停止参与和报告不利体验的方式。
- 使用研究专用设备/账户；不收集姓名、学校、班级、联系方式或自由输入身份信息。
- 先完成成人走查、键盘走查、375/768/1024/1440 视口和 reduced-motion 验收。
- 冻结目标注册表、受控题包、主要指标、排除规则和分析计划；试点中途变更必须留版本记录。

### 5.2 建议的小规模可行性设计

- 目标不是证明学习效果，而是检验流程能否完成、数据能否解释、体验是否安全。
- 使用合成或公开材料，先进行 5–8 个短会话的成人模拟，再决定是否进入经批准的学习者试点。
- 每次 10–15 分钟；避免让每日连续、奖励或研究人员暗示影响成绩。
- 对错反馈后允许随时停止；试点结束提供本地删除和导出检查。

### 5.3 预注册主要指标

1. 24 小时保持探测正确率：仅 `delayed-independent`、首次到期尝试。
2. 7 天保持探测正确率：仅 `delayed-independent`、首次到期尝试。
3. 新语境迁移正确率：仅 `transfer-independent` 且审核通过。
4. 到期复习完成率：分母为观察窗内真实到期项目，分子为完成项目。
5. 提示依赖率：有机会请求提示的作答中实际使用提示的比例。

次要指标：任务完成、主动退出、会话时长、活跃学习日、主观清晰度与不适报告。XP、金币、连击和总时长不得替代主要学习指标。

### 5.4 分析与停止规则

- 同时报告分子/分母、缺失与退出，不只报告百分比。
- 不把同一题重复、即时修复、未审核题或未知目标计入延迟/迁移分母。
- 不做排行榜、能力诊断或儿童间公开比较。
- 任何隐私泄露、无法退出、持续不适、明显诱导性奖励或错误答案键都触发暂停。
- 样本量不足时只报告可行性和观察，不宣称显著提升、因果效果或可推广性。

## 6. 仓库验收证据

- lint、Jest、生产构建、生成内容评测和 Playwright E2E 必须全部通过。
- 数据库迁移测试必须证明旧掌握度不会被继承成新证据。
- 教师审核测试必须证明未审核题不可获得测量资格。
- 浏览器验收必须使用合成数据并在完成后关闭本地服务。

“试点准备完成”只表示上述产品、隐私、测量和技术门槛可供审查；只有经批准并真实执行的研究才能产生学习成效证据。
