# Capture Checklist

Use this when recording videos or taking screenshots for Kimi.

## Recommended Input Mix

Best package:

- 7 live URLs.
- 7 desktop hero screenshots.
- 2 mobile screenshots.
- 3 short videos.

Avoid sending 20+ websites. More references will dilute the target style.

## Screenshot Set

Capture these as still images:

1. Current Word Quest `/project#hero`.
   - Purpose: baseline before redesign.
   - Capture desktop 1440x900 and mobile 390x844.

2. Linear hero and first product section.
   - Purpose: premium product-system layout.
   - Capture hero and one mid-page screenshot with product UI.

3. Cursor hero and AI agent/demo section.
   - Purpose: AI-native product storytelling.
   - Capture hero and one product-demo screenshot.

4. Raycast hero and feature cards.
   - Purpose: dark command-center mood.
   - Capture hero and one feature section.

5. Framer hero and one animated transition state.
   - Purpose: motion rhythm and product-stage feeling.

6. Apple iPad Pro first reveal.
   - Purpose: product object as hero.

7. Anthropic homepage trust/editorial section.
   - Purpose: serious AI and responsible tone.

8. Codédex hero or journey section.
   - Purpose: learning-adventure tone.

## Video Set

Record only short clips, ideally 30-60 seconds each.

### Video 1: Linear

Start at hero. Scroll slowly through:

- Hero headline.
- First product interface.
- First two numbered product sections.

Kimi should learn:

- How product screenshots anchor the page.
- How long sections stay structured.
- How text and UI screenshots alternate.

### Video 2: Cursor

Start at hero. Scroll through:

- Hero.
- Product demo.
- AI agent/workflow section.
- Social proof if visible.

Kimi should learn:

- How to present AI as a workflow, not a magic claim.
- How to stage product UI as a live demonstration.

### Video 3: Apple iPad Pro or Framer

Choose one:

- Apple if you want a product-reveal hero.
- Framer if you want more motion and scroll transitions.

Kimi should learn:

- How to use motion sparingly.
- How to make the primary object appear early and confidently.

## Optional Video 4: Raycast

Use only if the redesign should lean darker and more command-center-like.

Kimi should learn:

- High-contrast dark UI.
- Strong accent color used sparingly.
- Compact feature rhythm.

## What Not To Capture

- Pricing pages.
- Blog pages.
- Cookie modals.
- Login screens.
- Long footer areas.
- Anything that is mostly brand-only without product structure.

## Naming Convention

Use names like:

- `wordquest-current-desktop-hero.png`
- `linear-desktop-hero.png`
- `linear-product-section.png`
- `cursor-desktop-hero.png`
- `raycast-dark-hero.png`
- `apple-product-reveal.mp4`
- `framer-scroll-rhythm.mp4`

Keep videos under 100 MB if possible.
